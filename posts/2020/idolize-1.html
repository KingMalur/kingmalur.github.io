DevLog #1 - Idolize!

Hi 👋 and welcome to DevLog #1 of my first microGame Idolize!

In this game you control a wannabe influencer and pollute the internet with your thoughts. You have to fill out the blanks in your posts to collect and cultivate followers.

I started by creating a rough concept on how the game should work:

  1. Generate a post
  2. Generate three possible answers
  3. Select answer
  4. See effect

I then started to think about the configuration and data files and thought that Json should work ok.

After that I came up with a basic structure for my files. Here you can see my concept data structure for the posts. Each post consists of a body and can get one or multiple answers (here: option) depending on the pick value.

            
{
  "body": [
    {
      "id": 1,
      "text": "Ughh, I'm so booored! Ask me anything about my {_}!",
      "pick": 1,
      "value": 5,
      "category": "Normal",
    },
  ],
  "option": [
    {
      "id": 1,
      "text": "Dog",
      "value": 5,
      "category": "Normal"
    },
  ],
}

You can score combos if the categories match!

But my SaveData objects look rather boring:

            
{
    "data": [
        {
            "name": "Followers",
            "value": 42,
            "type": "Integer"
        },
    ]
}

But another interesting file to come up with was the message history:

            
{
  "history": [
    {
      "bodyId": 1,
      "optionId":
        [
          1,
        ],
      "followersGained": 15
    },
  ],
}

Some posts allow multiple options to be selected!

I wanted to a lightweight file to save every message choice the player selected at some point but without some implementation done right now I don’t know if that file’s completed yet.
